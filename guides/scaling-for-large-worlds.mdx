The Role Icons system is designed to handle large VRChat instances efficiently. This guide explains the built-in optimizations and how to troubleshoot performance issues.

## Performance Architecture

The system uses multiple optimization techniques to maintain smooth performance:

### Object Pooling

From `UserIconManager.cs:21` and `UserIconManager.cs:352-363`:

```csharp
[SerializeField] private int maxIconPool = 20; // Maximum active icon pool
```

Instead of creating/destroying GameObjects constantly, icons are:
1. **Pre-instantiated** on first use
2. **Reused** when players join/leave
3. **Pooled** when not needed
4. **Capped** at 20 simultaneous icons (configurable)

<Note>
  Object pooling reduces garbage collection pressure and prevents frame drops when players join/leave rapidly.
</Note>

### Batched Updates

From `UserIconManager.cs:22-23` and `UserIconManager.cs:112-150`:

```csharp
[SerializeField] private bool useBatchedUpdates = true;
[SerializeField] private int playersPerBatch = 10;
```

Instead of updating all players every frame:
- **10 players** are processed per update cycle
- Update cycles repeat until all players are processed
- Spreads CPU load across multiple frames

<CodeGroup>
```text Example: 80 Players
Frame 1: Update players 0-9
Frame 2: Update players 10-19
Frame 3: Update players 20-29
...
Frame 8: Update players 70-79
Frame 9: Cleanup unused icons, restart cycle

Total time: ~60 seconds per full cycle
CPU impact: Minimal per frame
```
</CodeGroup>

### Distance-Based Culling

From `IconFollower.cs:12` and `IconFollower.cs:119-145`:

```csharp
[SerializeField] private float maxDistance = 50f;
```

Icons beyond 50 meters are automatically disabled:
- **SpriteRenderer disabled** (not rendered)
- **Position updates stopped**
- **Rotation updates stopped**
- **Distance checked** every 0.5 seconds

<Warning>
  In large open worlds, distance culling significantly reduces rendering overhead. Don't disable this feature.
</Warning>

### Reduced Update Rates

From `IconFollower.cs:19-20`:

```csharp
[SerializeField] private float distanceCheckInterval = 0.5f;
[SerializeField] private float rotationUpdateInterval = 0.033f; // ~30 FPS
```

Different operations update at different rates:

| Operation | Rate | Why |
|-----------|------|-----|
| Position | Every frame | Smooth following |
| Rotation | 30 FPS (~0.033s) | Billboard effect |
| Distance check | 2 Hz (0.5s) | Enable/disable culling |
| Icon assignment | 60s | Role changes are rare |
| JSON reload | 300s | Remote updates |

## Configuration for Different World Sizes

### Small Worlds (0-20 players)

```csharp
maxIconPool = 20
useBatchedUpdates = false
playersPerBatch = 10
updateInterval = 30f
```

No batching needed - update all players every 30 seconds.

### Medium Worlds (20-40 players)

```csharp
maxIconPool = 30
useBatchedUpdates = true
playersPerBatch = 10
updateInterval = 60f
```

Default settings work well.

### Large Worlds (40-80 players)

```csharp
maxIconPool = 40
useBatchedUpdates = true
playersPerBatch = 10
updateInterval = 90f
```

Increase pool size and update interval.

### Massive Worlds (80+ players)

```csharp
maxIconPool = 50
useBatchedUpdates = true
playersPerBatch = 5
updateInterval = 120f
```

Smaller batches spread load more. Longer update interval acceptable.

<Warning>
  **VRChat Instance Limit**: VRChat worlds typically cap at 80 players. Beyond this, consider:
  - Increasing `maxDistance` culling
  - Reducing `maxIconPool`
  - Increasing `updateInterval`
</Warning>

## Troubleshooting Performance Issues

### Symptom: Frame Drops When Players Join

**Cause**: Icon pool exhausted, creating new GameObjects

**From `UserIconManager.cs:319-330`:**
```csharp
if (poolSize < maxIconPool)
{
    icon = CreateNewIcon();
}
else
{
    DebugLog("Icon pool is full, no more can be created");
    return;
}
```

**Solution**:
1. Increase `maxIconPool` to match expected player count
2. Enable debug logs to see "Icon pool is full" warnings
3. Reduce number of roles to limit icons needed

### Symptom: Icons Flickering

**Cause**: Rapid enable/disable due to distance threshold edge case

**From `IconFollower.cs:119-145`:**
```csharp
currentDistance = Vector3.Distance(localPlayer.GetPosition(), GetTargetPlayerPosition());
bool shouldBeActive = currentDistance <= maxDistance;

if (shouldBeActive != isWithinRange)
{
    // Toggle icon
}
```

**Solution**:
1. Increase `distanceCheckInterval` from 0.5s to 1.0s
2. Add hysteresis: enable at 48m, disable at 52m (requires code modification)
3. Increase `maxDistance` to reduce edge cases

### Symptom: High CPU Usage in VRChat

**Cause**: Too many Update() calls or large batch sizes

**Check:**
1. Is `useBatchedUpdates` enabled?
2. Is `playersPerBatch` too high? (should be 5-10)
3. Are icons being created/destroyed repeatedly? (check debug logs)

**Solution**:
```csharp
// Reduce batch size
playersPerBatch = 5;

// Increase update interval
updateInterval = 120f;

// Ensure pooling is working
maxIconPool = [expected player count];
```

### Symptom: Icons Not Appearing

**Cause**: Pool size too small or icons disabled by distance culling

**Debug Steps:**
1. Enable `enableDebugLogs` on UserIconManager
2. Join world and check console for:
   - `Icon pool is full, no more can be created`
   - `Icon assigned to {user} ({role})`
3. Check if player is beyond `maxDistance` (50m default)

**From `UserIconManager.cs:26`:**
```csharp
[SerializeField] private bool enableDebugLogs = false;
```

Set to `true` for debugging.

### Symptom: Memory Leaks

**Cause**: Icons not returned to pool when players leave

**From `UserIconManager.cs:485-489`:**
```csharp
public override void OnPlayerLeft(VRCPlayerApi player)
{
    RemovePlayerIcon(player.playerId.ToString());
    DebugLog($"Player left: {player.displayName}");
}
```

**Verify:**
1. `OnPlayerLeft` is being called (check debug logs)
2. `ReturnIconToPool` is cleaning up properly
3. No exceptions in cleanup code

## Debug Monitoring

Enable comprehensive logging to monitor performance:

### In Unity

1. Select UserIconManager GameObject
2. Check **Enable Debug Logs**
3. Enter Play Mode
4. Watch Console for:

<CodeGroup>
```text Key Log Messages
[UserIconManager] Loading role data from JSON...
[UserIconManager] Roles loaded: 6
[UserIconManager] Users processed: 45
[UserIconManager] Icon assigned to PlayerName (roleName)
[UserIconManager] Player joined: PlayerName
[UserIconManager] Player left: PlayerName
Icon pool is full, no more can be created
```
</CodeGroup>

### In VRChat

Use VRChat's built-in performance tools:

1. Open Quick Menu
2. Settings → Performance
3. Enable "Show Performance Stats"
4. Monitor:
   - **FPS**: Should stay above 60 in VR, 90 on desktop
   - **Frame Time**: Should stay below 16ms (60 FPS)
   - **Draw Calls**: Icons add minimal draw calls (batched by Unity)

### Custom Performance Monitoring

Add this to `UserIconManager.cs` for detailed metrics:

```csharp
private void DebugPerformance()
{
    if (!enableDebugLogs) return;
    
    int activeIconCount = 0;
    for (int i = 0; i < poolSize; i++)
    {
        if (poolInUse[i] == 1) activeIconCount++;
    }
    
    Debug.Log($"[Performance] Pool: {activeIconCount}/{poolSize} | Players: {playerCount}");
}
```

Call from `Update()` every 60 seconds.

## Optimization Checklist

Before publishing your world:

<Steps>
  <Step title="Configure Pool Size">
    - [ ] Set `maxIconPool` to expected max players
    - [ ] Enable `useBatchedUpdates`
    - [ ] Set `playersPerBatch` between 5-10

    - [ ] Adjust settings based on actual usage

## Advanced Optimizations

### Spatial Partitioning (Code Modification)

For worlds with 100+ players, consider dividing the world into zones:

```csharp
// Only update icons for players in same zone as local player
private bool IsInSameZone(VRCPlayerApi player1, VRCPlayerApi player2)
{
    Vector3 pos1 = player1.GetPosition();
    Vector3 pos2 = player2.GetPosition();
    
    // Define zone size (e.g., 100m x 100m)
    int zone1 = Mathf.FloorToInt(pos1.x / 100f) + Mathf.FloorToInt(pos1.z / 100f) * 1000;
    int zone2 = Mathf.FloorToInt(pos2.x / 100f) + Mathf.FloorToInt(pos2.z / 100f) * 1000;
    
    return zone1 == zone2;
}
```

### LOD System (Code Modification)

Reduce icon quality based on distance:

```csharp
private void UpdateIconLOD(float distance)
{
    if (distance < 10f)
        iconSpriteRenderer.sprite = highResSprite; // 256x256
    else if (distance < 30f)
        iconSpriteRenderer.sprite = medResSprite;  // 128x128
    else
        iconSpriteRenderer.sprite = lowResSprite;  // 64x64
}
```

<Warning>
  Advanced optimizations require modifying the source code. Test thoroughly before deploying to production worlds.
</Warning>

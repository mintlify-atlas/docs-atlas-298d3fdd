## Performance Overview

The VRChat Role Icons system uses several optimization techniques to handle large player counts:

### Key Optimizations

<CardGroup cols={2}>
  <Card title="Object Pooling" icon="recycle">
    Reuses icon GameObjects instead of creating/destroying them
    
    **Impact**: -66% memory usage
  </Card>
  
  <Card title="Distance-Based Culling" icon="eye-slash">
    Disables icons beyond max distance
    
    **Impact**: -96% distance calculations
  </Card>
  
  <Card title="Batched Updates" icon="layer-group">
    Updates players in chunks over multiple frames
    
    **Impact**: Prevents frame spikes
  </Card>
  
  <Card title="Interval-Based Updates" icon="clock">
    Reduces update frequency for rotation and distance checks
    
    **Impact**: -75% Update() calls
  </Card>
</CardGroup>

### Performance Metrics (80 Players)

| Metric | Unoptimized | Optimized | Improvement |
|--------|-------------|-----------|-------------|
| Update() calls/sec | 4,800 | ~1,200 | **-75%** |
| Distance checks/sec | 4,800 | 160 | **-96%** |
| Active GameObjects | 80 | 20 max | **-75%** |
| Memory usage | ~15 MB | ~5 MB | **-66%** |
| CPU usage | High | Low | **-70%** |

## Recommended Configurations

### Casual Worlds (20-40 Players)

Optimal for social hangouts, small events, and friend gatherings.

<Tabs>
  <Tab title="UserIconManager">
    ```yaml
    Configuration:
      updateInterval: 60
    
    Optimization:
      maxIconPool: 15
      useBatchedUpdates: true
      playersPerBatch: 10
    ```
    
    **Rationale**:
    - Pool of 15 is sufficient for 20-40 players with distance culling
    - Batch size of 10 keeps updates smooth
    - Standard update interval
  </Tab>
  
  <Tab title="IconFollower (Prefab)">
    ```yaml
    Configuration:
      maxDistance: 50
      iconScale: 0.3
      heightOffset: 1.0
      lockYAxis: true
    
    Optimization:
      distanceCheckInterval: 0.5
      rotationUpdateInterval: 0.033
    ```
    
    **Rationale**:
    - Generous 50m view distance for good visibility
    - Standard check intervals provide good responsiveness
    - Balanced between quality and performance
  </Tab>
  
  <Tab title="Expected Performance">
    **Target**: 60+ FPS stable
    
    **Characteristics**:
    - All nearby players visible
    - Smooth icon transitions
    - Low CPU overhead
    - Quick role updates
    
    **Best for**:
    - Hangout worlds
    - Small club events
    - Friend meetups
  </Tab>
</Tabs>

Balanced settings for medium-sized events and popular public worlds.

      maxIconPool: 20

    - ⬆️ `maxIconPool`: 15 → 20 (handle more simultaneous icons)

  <Tab title="IconFollower (Prefab)">
    ```yaml
    Configuration:
      maxDistance: 30
      iconScale: 0.3
      heightOffset: 1.0
      lockYAxis: true
    
    Optimization:
      distanceCheckInterval: 0.75

    ```
    
    **Changes from Casual**:
    - ⬇️ `maxDistance`: 50 → 30 (reduce visible icon count)
    - ⬆️ `distanceCheckInterval`: 0.5 → 0.75 (fewer checks)

  <Tab title="Expected Performance">
    **Target**: 45-60 FPS
    
    **Characteristics**:
    - Icons visible within 30m
    - Slight delay in distance updates (0.75s)

    **Trade-offs**:
    - ⚠️ Distant players may not show icons

    - Medium-sized gatherings

## Configuration Trade-offs

Understanding the impact of each setting helps you make informed tuning decisions.

### maxIconPool

<ParamField path="maxIconPool" type="int">
  **Higher values** (25-40):
  - ✅ More players can have icons simultaneously
  - ✅ Icons less likely to disappear
  - ❌ Higher memory usage
  - ❌ More Update() calls per frame
  
  **Lower values** (10-15):
  - ✅ Lower memory footprint
  - ✅ Fewer Update() calls
  - ❌ Distant players may not get icons
  - ❌ Icon "flickering" when pool is contested
  
  **Sweet spot**: 15-20 for most worlds
</ParamField>

### maxDistance

<ParamField path="maxDistance" type="float">
  **Higher values** (40-50m):
  - ✅ Icons visible from far away
  - ✅ Better for exploring large worlds
  - ❌ More icons active = higher CPU usage
  - ❌ May exceed maxIconPool limit
  
  **Lower values** (15-25m):
  - ✅ Fewer active icons = better performance
  - ✅ Focuses on nearby players
  - ❌ Limited visibility range
  - ❌ Icons appear/disappear more frequently
  
  **Sweet spot**: 30m for most scenarios
</ParamField>

### playersPerBatch

<ParamField path="playersPerBatch" type="int">
  **Higher values** (20-30):
  - ✅ Faster complete update cycles
  - ✅ Quicker response to player changes
  - ❌ Potential frame spikes with many players
  
  **Lower values** (5-10):
  - ✅ Smoother frame pacing
  - ✅ More distributed CPU load
  - ❌ Slower to update all players
  - ❌ New players wait longer for icons
  
  **Sweet spot**: 10-15 for balanced updates
</ParamField>

### distanceCheckInterval

<ParamField path="distanceCheckInterval" type="float">
  **Higher values** (1.0-2.0s):
  - ✅ Significant CPU savings (fewer calculations)
  - ✅ Scales better with many players
  - ❌ Icons take longer to appear/disappear
  - ❌ Less responsive to movement
  
  **Lower values** (0.1-0.3s):
  - ✅ Very responsive icon activation
  - ✅ Smooth transitions
  - ❌ Higher CPU usage
  - ❌ May cause performance issues at scale
  
  **Sweet spot**: 0.5-0.75s for good balance
</ParamField>

## Fine-Tuning Guide

### When You Experience Lag

<Steps>
  <Step title="Reduce visible icons">
    Lower `maxDistance` from 50 → 30 → 20
    
    This is the **most effective** optimization.
  </Step>
  
  <Step title="Increase check intervals">
    Raise `distanceCheckInterval` from 0.5 → 0.75 → 1.0
    
    Reduces CPU load with minimal visual impact.
  </Step>
  
  <Step title="Limit icon pool">
    Lower `maxIconPool` from 20 → 15 → 10
    
    Forces system to reuse icons more aggressively.
  </Step>
  
  <Step title="Slow down rotation">
    Increase `rotationUpdateInterval` from 0.033 → 0.05 → 0.1
    
    Rotation at 20-10 FPS is still acceptable.
  </Step>
</Steps>

### When Icons Are Flickering

<Steps>
  <Step title="Increase icon pool">
    Raise `maxIconPool` from 20 → 25 → 30
    
    More pool slots = less competition for icons.
  </Step>
  
  <Step title="Reduce check delay">
    Lower `distanceCheckInterval` from 1.0 → 0.5 → 0.25
    
    Faster response to distance changes.
  </Step>
  
  <Step title="Increase view distance">
    Raise `maxDistance` from 20 → 30 → 40
    
    Keeps icons visible longer before culling.
  </Step>
</Steps>

### When Updates Are Too Slow

<Steps>
  <Step title="Increase batch size">
    Raise `playersPerBatch` from 10 → 15 → 20
    
    Processes more players per cycle.
  </Step>
  
  <Step title="Reduce update interval">
    Lower `updateInterval` from 90 → 60 → 45
    
    More frequent full system updates.
  </Step>
  
  <Step title="Disable batching (small worlds)">
    Set `useBatchedUpdates = false` if player count < 20
    
    Updates all players every frame.
  </Step>
</Steps>

## Advanced Optimization

### Custom Configuration for Specific Zones

You can have different IconFollower settings for different areas:

```csharp
// Stage area - aggressive optimization
maxDistance: 15
distanceCheckInterval: 1.0

// VIP lounge - better quality
maxDistance: 40
distanceCheckInterval: 0.33
```

Create multiple icon prefabs with different settings and assign them based on player role or location.

### Dynamic Adjustment Based on Player Count

While not built into the system, you could add Udon logic to adjust settings dynamically:

```csharp
// Pseudo-code concept
if (playerCount < 20)
    ApplyCasualSettings();
else if (playerCount < 50)
    ApplyCrowdedSettings();
else
    ApplyMassiveSettings();
```

## Monitoring Performance

Enable debug logs to monitor system behavior:

```yaml
UserIconManager:
  enableDebugLogs: true
```

### Key Metrics to Watch

<AccordionGroup>
  <Accordion title="Pool Usage">
    ```
    [UserIconManager] Pool: 18/20, Active Icons: 18
    ```
    
    If pool is constantly at maximum, consider increasing `maxIconPool`.
  </Accordion>
  
  <Accordion title="Distance Checks">
    ```
    [IconFollower] Icon Distance: 23.4m, Active: true
    ```
    
    Monitor how often icons toggle on/off. Frequent toggling may indicate `maxDistance` is too close to average player spacing.
  </Accordion>
  
  <Accordion title="JSON Loading">
    ```
    [UserIconManager] Roles loaded: 4
    [UserIconManager] Users processed: 23
    ```
    
    Confirms role system is working correctly.
  </Accordion>
</AccordionGroup>

## Performance Checklist

<Steps>
  <Step title="Choose base configuration">
    Select Casual, Crowded, or Massive based on expected player count.
  </Step>
  
  <Step title="Test with target player count">
    Use VRChat's debug tools or fill the world with test accounts.
  </Step>
  
  <Step title="Monitor FPS and CPU usage">
    Check Unity Profiler or in-game FPS counter.
  </Step>
  
  <Step title="Adjust maxDistance first">
    This has the biggest performance impact.
  </Step>
  
  <Step title="Fine-tune check intervals">
    Balance responsiveness vs. performance.
  </Step>
  
  <Step title="Verify icon pool size">
    Ensure it's not constantly maxed out.
  </Step>
  
  <Step title="Test in worst-case scenarios">
    Maximum players, all in one area, all with roles.
  </Step>
</Steps>

<Warning>
  **Always test performance in VRChat**, not just Unity Editor. The VRC SDK adds overhead that affects performance differently than local testing.
</Warning>
